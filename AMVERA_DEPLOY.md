# Перенос Добро.Медиа на Amvera

Эти файлы предназначены для параллельного запуска сайта на российском хостинге.
Ветка `main`, действующая версия Vercel и данные Supabase при переносе не изменяются.

## 1. Создание приложения

1. В Amvera создайте новый проект типа «Приложение».
2. Подключите репозиторий `mrkustov927-star/media-dobro`.
3. Для автоматического обновления выберите ветку `amvera-migration`.
4. Amvera прочитает `amvera.yml` и соберёт `Dockerfile.amvera`.
5. Контейнер приложения принимает соединения на порту `3000`.

## 2. Переменные и секреты

Скопируйте значения из действующего проекта Vercel, не публикуя их в GitHub.

Публичные переменные:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `NEXT_PUBLIC_VK_COMMUNITY_URL`
- `NEXT_PUBLIC_VK_MESSAGES_URL`

Секреты:

- `SUPABASE_SERVICE_ROLE_KEY`
- `ADMIN_PIN`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `RESEND_API_KEY`
- `ADMIN_EMAIL`
- `EMAIL_FROM`
- `VK_GROUP_TOKEN`
- `VK_ADMIN_PEER_ID`
- `VK_API_VERSION`

После изменения переменных перезапустите контейнер.

## 3. Сохранность данных

Новый сайт использует существующий проект Supabase. Все задания, заявки,
статусы, ссылки, время и комментарии останутся на месте.

Не выполняйте повторно файл `dobro-media-cabinet/supabase/schema.sql`:
в нём есть команды очистки таблиц.

## 4. Проверка до переключения адреса

Проверяйте Amvera и Vercel параллельно:

1. Главная страница открывается без VPN.
2. Календарь показывает существующие данные.
3. Можно взять тестовую активность.
4. Можно сдать ссылку на материал.
5. Открывается `/admin`.
6. Принимается действующий PIN.
7. Сохраняются статусы и комментарии.
8. Работают уведомления ВКонтакте, Telegram и электронной почты.

Только после этих проверок подключайте основной домен к Amvera.
Vercel следует оставить резервной версией до завершения переноса.
