# Инструкция по запуску кабинета Добро.Медиа

## 1. Что уже лежит в GitHub

Папка проекта: `dobro-media-cabinet`.

Внутри есть:

- сайт для ребят;
- страница администратора `/admin`;
- SQL-схема Supabase;
- календарь заданий на июль;
- API для взятия активности, сдачи материала и админского редактирования;
- пример переменных окружения `.env.example`.

## 2. Создать Supabase

1. Открой Supabase.
2. Создай новый проект.
3. Открой раздел SQL Editor.
4. Скопируй и выполни файл `supabase/schema.sql`.
5. Если живые обновления не заработают сразу, выполни файл `supabase/realtime.sql` или включи Realtime для таблиц `activities` и `assignments` в интерфейсе Supabase.

## 3. Взять ключи Supabase

В Supabase открой Project Settings → API.

Нужны значения:

- Project URL;
- anon public key;
- service_role key.

`service_role key` нельзя показывать ребятам и нельзя вставлять в публичный код.

## 4. Подключить Vercel

1. Открой Vercel.
2. Создай новый проект из GitHub-репозитория `media-dobro`.
3. В настройках проекта укажи Root Directory: `dobro-media-cabinet`.
4. Framework Preset: Next.js.
5. Добавь переменные окружения.

## 5. Переменные окружения Vercel

Добавь:

```text
NEXT_PUBLIC_SUPABASE_URL=Project URL из Supabase
NEXT_PUBLIC_SUPABASE_ANON_KEY=anon public key из Supabase
SUPABASE_SERVICE_ROLE_KEY=service_role key из Supabase
ADMIN_PIN=любой секретный PIN для Кустова Евгения Валерьевича
```

После добавления переменных нужно сделать redeploy.

## 6. Как пользоваться ребятам

1. Открыть сайт.
2. Выбрать дату и активность в календаре.
3. Прочитать карточку задания.
4. Нажать «Взять активность».
5. Выполнить задание.
6. Нажать «Сдать материал».

Ребята видят, кто уже взял активность и какой у неё статус.

## 7. Как пользоваться администратору

1. Открыть `/admin`.
2. Ввести `ADMIN_PIN`.
3. Менять статусы заявок:
   - Взято в работу;
   - Материал сдан;
   - На доработке;
   - Проверено;
   - Зачтено;
   - Отменено.
4. Указывать потраченное время.
5. Оставлять комментарий.
6. Редактировать активности календаря.

Все сохранённые изменения становятся видны ребятам на сайте.

## 8. Важное правило

Материалы не публикуются ребятами самостоятельно. Готовые посты, фото, видео и ролики отправляются Кустову Евгению Валерьевичу на проверку.
