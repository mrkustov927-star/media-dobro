# Добро.Медиа — кабинет медиа-волонтёра

Это версия 2.0 для проекта «Добро.Медиа»: не статичная страница, а рабочий кабинет с интерактивным календарём, видимостью занятых активностей и админ-панелью для Кустова Евгения Валерьевича.

## Что будет у ребят

- календарь на июль;
- карточка задания по каждой активности;
- понятные подсказки: о чём событие, что делать, как делать, что собрать и что отправить;
- кнопка «Взять активность»;
- кнопка «Сдать материал»;
- видимость, кто уже взял задание и какой у него статус.

## Что будет у администратора

- страница `/admin`;
- изменение статуса активности волонтёра;
- учёт потраченного времени;
- комментарии администратора;
- редактирование календаря прямо на сайте;
- добавление новых активностей;
- скрытие или показ активностей;
- живое обновление данных через Supabase.

## Стек

- Next.js;
- Supabase;
- Vercel;
- GitHub.

## Основные файлы

- `app/page.tsx` — кабинет для ребят;
- `app/admin/page.tsx` — админ-панель;
- `app/api/claim/route.ts` — взять активность;
- `app/api/submit/route.ts` — сдать материал;
- `app/api/admin/update-assignment/route.ts` — админское изменение заявки;
- `app/api/admin/update-activity/route.ts` — админское изменение активности;
- `app/api/admin/create-activity/route.ts` — добавление активности;
- `supabase/schema.sql` — таблицы и стартовый календарь;
- `supabase/realtime.sql` — включение живых обновлений;
- `SETUP.md` — пошаговая инструкция запуска.

## Следующие шаги

1. Создать проект Supabase.
2. Выполнить SQL из `supabase/schema.sql`.
3. При необходимости выполнить `supabase/realtime.sql`.
4. Создать проект Vercel с корневой папкой `dobro-media-cabinet`.
5. Добавить переменные окружения из `.env.example`.
6. Открыть сайт и проверить календарь.
7. Открыть `/admin`, ввести `ADMIN_PIN` и проверить редактирование.
